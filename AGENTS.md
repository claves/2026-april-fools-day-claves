# エイプリルフール企画：くまちょんショップ

## BDD開発における絶対ルール

### 仕様の正（Single Source of Truth）

**`specs/*.feature` ファイルが仕様の絶対的な正である。**

- `specs/product-list.feature`: 商品一覧ページの仕様
- `specs/purchase.feature`: 購入画面の仕様

### 絶対遵守事項

1. **featureに書かれていない機能は実装しない**
   - featureに記載のないUI要素、動作、機能は追加禁止
   - 「あったほうが良い」「便利そう」という理由での追加は禁止

2. **featureと矛盾する実装は禁止**
   - featureのシナリオと異なる動作になる実装は禁止
   - 実装がfeatureと矛盾した場合、実装を修正する（featureを変えない）

3. **featureの変更は事前承認が必要**
   - featureファイルの追加・変更・削除は必ずユーザーの承認を得る
   - AIが勝手にfeatureを書き換えてはならない

4. **実装前にテスト失敗を確認する**
   - 新規実装・修正の前に `npm test` で該当シナリオが失敗することを確認
   - 失敗を確認せずに実装を始めることは禁止

5. **実装後にテスト成功を確認する**
   - 実装後に `npm test` で該当シナリオが成功することを確認
   - テストが通らない状態で作業完了としない

## プロジェクト概要

会社のマスコットキャラクター「くまちょん」がAIにECサイト制作を任せたという設定の、**意図的に使いにくいECサイト**。エイプリルフール用のネタコンテンツ。

## コンセプト

- **くまちょん**（会社のマスコット・クマのキャラクター）がECサイトをAIに丸投げした
- AIが作った結果、盛大に失敗した
- くまちょんはAIを信じていたため把握していない、という言い訳スタンス
- **くまちょんはイラスト・絵文字を使わず、名前のみで表現する**（🐻は使わない）

## ファイル構成

```
2026_april/
└── index.html   # フロントのみ・単一ファイル完結（購入処理の実装なし）
```

## 画面構成

1. **商品一覧ページ**（`#page-list`）
2. **購入画面**（`#page-purchase`）

ページ遷移はJavaScriptで制御（SPA風）。

## デザイン方針

- パステル・ポップなかわいい系デザイン
- カラーパレット：ピンク系メイン・イエロー・ミント・ラベンダー
- フォント：M PLUS Rounded 1c（角丸）+ Kaisei Decol（和風セリフ）
- ドット背景、角丸カード、ピル型ボタン

## ナレーションのトーン

- くまちょんが発注者、AIが制作者（失敗）
- 「くまちょんも把握していません」「くまちょんはAIを信じています」「くまちょんは責任を負いません」
- AIへのクレームはくまちょんに言わないでください、というスタンス
